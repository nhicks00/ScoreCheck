The live path — video in, score out

Layer 0Data substrateBuilt

NAS corpus of owned VODs; scorebug OCR + template classifier turn every stream into tiered rally labels (gold / silver / excluded by human-scoring noise fingerprints); overlay-masked clips; match-level train / search / selection / test splits.

▼
Layer 2Event & state inference — deterministic code, not ML

Rally state machine (SERVICE onset starts, ball-death + NO-PLAY ends) → side-out attribution (whoever serves next won the last rally) → FIVB rules reducer (26 behavioral tests) as the sole score authority, with side-switch checksums every 7/5 points. Every event carries confidence; low confidence goes to a one-tap human confirm in ScoreCheck. No model ever writes a score directly.

▼
Layer 3Products

Score keeping: offline replay → live shadow → assistive confirm → autonomous with corrections. Analytics on the same event log: touch classification, rally lengths, serve speed, placement maps, auto-cut highlights. Live path taps MediaMTX upstream of the 720p normalization; CoreML/ONNX exports run real-time on Apple Silicon.

The flywheel — how it gets better without a labeling team

Campaign opensNathan approves goal metric, $ cap, stop rules. Baseline runs first; noise floor measured.
Train on ModalOne hypothesis per git branch, config-hashed, $2–5 per run.
Locked harnessGold-tier winner accuracy + timing-immune sequence metric on splits the agent can't read or edit.
Ledger + verdictAppend-only JSONL in git. Keep only wins above baseline + noise.
Mine failuresModel-vs-scorebug disagreement pulls lookalike windows from 157 h of corpus.
Propose nextAgent writes the morning summary and the next experiment. Human steers at boundaries only.

Why we can run this loop at all: the scorebug gives a free, independent goal signal on real matches — graded for human-entry noise, cross-checked later by side-out physics, and anchored by a small human-verified kernel. NVIDIA's autoresearch pattern, our verifiable reward.
